# 轻截

一个使用 C# / .NET 8 / WinForms 编写的 Windows 桌面截图工具。程序可以常驻系统托盘，在后台监听全局快捷键；默认按 `Ctrl + Shift + X` 开始截图。

## 已实现功能

- 左侧分页、右侧内容的可视化工作台，左上角明确显示当前版本号，并集中提供信息粘贴、截图设置、保存、编辑和截图浏览入口。
- 使用轻截专属的深蓝取景框与切割闪光图标，内置 16–256 像素多尺寸资源，统一应用于 EXE 文件、窗口标题栏、任务栏和系统托盘。
- “截图设置”统一管理全局截图快捷键、开机自动启动、手动启动后最小化、截图前提示和主界面显示行为；开机启动使用当前用户启动项，不需要管理员权限，并会安静进入系统托盘。“保存路径”独立管理保存目录与图片命名，可选择日期时间、保存目录递增序号或图片内文字。
- 保存路径支持从资源管理器拖入单个文件夹，自动引用其完整路径；切换到新路径时会询问是否一并移动旧目录中的图片，同名图片不会被覆盖。
- “查看截图”页会读取保存目录中的最近截图，支持刷新、打开目录和双击查看原图。
- 可挂接截图保存文件夹，`Ctrl + S` 按用户选择的规则保存 PNG，并自动复制最终图片到剪贴板。“图片内文字”模式会组合最终截图范围内的文字编辑元素，清理 Windows 文件名非法字符后用于命名；没有可用文字时回退到日期时间，重名时自动追加序号。
- 截图或录屏保存后在当前屏幕右下角显示约 6 秒成功通知；点击通知会打开目录并选中刚保存的图片或 MP4 文件。
- 默认在开始下一次截图前自动关闭仍在显示的保存通知，避免通知被截入画面或妨碍连续操作；可在“截图设置”页关闭该开关，以便保留通知演示点击跳转功能。
- “截图设置”页同时提供“截图时隐藏轻截主界面”开关，默认关闭，因此可以直接把轻截工作台拍进宣传截图。开启后主窗口会在抓屏前先排除捕获、立即透明并隐藏，淡出动画不会再残留在截图中。
- 关闭主窗口后继续在系统托盘后台工作，托盘菜单可截图、打开设置或退出。
- “软件更新”页可直接检查 GitHub Releases 的最新正式版；发现新版本后会自动选择轻量包或自带运行库的便携包，下载并核对 GitHub SHA-256，退出当前进程后原地替换、失败回滚并自动重启，不需要用户重新下载和解压安装包。已永久删除的插件不会在整包更新时被重新安装。
- 支持多显示器虚拟桌面截图。
- 鼠标悬停时自动识别并高亮顶层窗口，左键单击即可确认该窗口。
- 按住左键并拖动会切换为自由框选；第一次按 `Ctrl + A` 选取鼠标所在显示器，第二次选取全部显示器；`Esc` 取消。
- 选区确认后可拖动四条边和四个角微调范围，光标会显示对应方向的双向箭头。
- 选区确认后可在选区内部按住鼠标右键拖动整个截图框，按下时显示四向移动光标。
- “信息粘贴”设置可切换贴纸跟随模式：图片、粘贴文字和工具栏文字可随截图框一起移动，也可保持屏幕位置；固定模式中的越界内容只会暂时隐藏，不会被删除。
- 临时编辑工具：矩形、椭圆、箭头、画笔、透明背景文字、马赛克、颜色、线宽、撤销。画笔和马赛克使用与实际作用范围一致的笔刷轮廓光标，可在设置中选择圆形或正方形。
- 图片、文字、方框、椭圆、箭头、画笔线条和马赛克统一使用 `Alt + 鼠标左键` 拖动；Alt 是临时移动模式，松开后会自动回到此前启用的绘图工具。单选可缩放元素后会显示四角与四边中点共八个手柄，边中手柄只调整对应一条边。
- 元素拖动或手柄缩放期间按住 `Ctrl` 会按固定像素步长变化，便于重复间距和尺寸排版；步长可在“图片修改”页配置。`Ctrl + 左键` 单击仍用于添加或取消多选，只有实际拖动后才进入步进移动。
- 编辑元素吸附默认开启，会在移动或单边缩放接近其他元素时，对齐双方的左/中/右或上/中/下参考线。共享编辑工具栏提供“吸附 开/关”按钮，快速双击 `Ctrl` 也可切换当前编辑会话；吸附开关默认值和吸附距离均可在“图片修改”页配置。
- 按住 `Alt` 并在编辑元素上滚动鼠标滚轮可旋转鼠标下的元素；向上为顺时针、向下为逆时针，每格旋转角度可在“图片修改”页配置。若鼠标下元素属于当前多选组，全部选中元素会同步旋转相同角度。
- 按住 `Ctrl` 并在编辑元素上滚动鼠标滚轮可缩放鼠标下的元素；向上放大、向下缩小，鼠标位置会作为缩放锚点。多选或 `Ctrl + A` 全选时，命中组内任一元素会缩放整个选中组并保持相对布局。
- 按住 `Ctrl` 并左键单击可添加或取消多个编辑元素的选中状态；使用 `Alt + 鼠标左键` 拖动任一已选元素会统一移动整组，按 `Delete` 可一次删除全部选中元素。
- 多个编辑元素重叠时，在同一小区域连续单击会按从上到下的顺序轮换选中，到底层后回到最上层；鼠标明显移到其他位置或元素集合变化后会重新从最上层开始。
- 单选工具栏文字或粘贴文字后双击，可重新进入文字编辑；提交时原地更新同一元素，保留原层级、颜色、类型和旋转角度，不会产生重复文字。
- 截图框内存在编辑元素且未启用绘图工具时，可按住鼠标左键拖出选择框，与选择框相交的编辑元素会被批量选中。
- `Ctrl + A` 会优先全选截图框中的全部编辑元素；元素已经全部选中后，继续按下会先把截图框扩展到鼠标所在显示器，再按一次扩展到全部显示器。没有编辑元素时直接从“当前显示器”开始这套扩展顺序。
- 截图框内已有编辑后，未选择工具时的左键不会误触重新框选；需要重划时按 `Ctrl + W` 启动重新框选，再按住左键拖动选择新区域。
- `Esc` 使用分层退出：先完成文字输入、取消元素选中、关闭绘图工具或结束截图框调整；仅在没有任何编辑状态时才关闭截图界面。
- 绘制工具按钮采用开关逻辑：单击启用，再次点击同一工具即可关闭。
- “图片修改”设置可配置编辑工具粗细上下限、元素旋转步进、笔刷光标外观、元素吸附默认开关、吸附距离和 `Ctrl` 拖动步长；截图时仅当鼠标悬停在“粗细”按钮上滚动，才会增减当前粗细。最后使用的粗细会自动保存，并在下次打开截图编辑器时恢复。粗细同时决定新建工具栏文字和粘贴文字的字号，编辑文字期间调整会即时生效。
- 选区确认后按 `Ctrl + V` 可粘贴剪贴板图片或文字；图片会作为贴纸自适应缩小，并支持拖动、四角等比缩放和四边中点单边缩放；粘贴文字与工具栏添加的文字也可选中拖动。
- 方框、椭圆、箭头、画笔线条和马赛克支持八手柄自由缩放，可分别调整宽度和高度；图片使用四角时保持等比缩放，使用边中手柄时只调整对应方向。
- 选区确认后可点击“长截图”：程序保留原选区边框，并在旁侧打开始终位于滚动目标上方的独立实时预览窗；预览窗可拖动、调整大小和最大化，在预览画面上滚轮可缩放、左键拖动可平移查看。用户在蓝色选区内用滚轮向上或向下滚动时，程序会周期采集中间帧并双向拼接，回访已采集区域不会重复添加内容。点击“完成并编辑”后，长图直接回填到原截图选区并复用普通截图工具栏和编辑系统；普通滚轮以鼠标位置缩放长图画面，`Ctrl + 滚轮` 缩放鼠标下的编辑元素，右键拖动选区内的长图画面可平移查看；还可继续使用矩形、椭圆、箭头、画笔、文字、马赛克、颜色、粗细、粘贴、撤销、多选、移动和缩放能力，再按原流程复制或保存。按 `Esc` 或点击“取消”会放弃本次长截图。“长截图”设置默认关闭安全校验，低置信度接缝或固定悬浮内容会按最可能位置继续拼接；需要严格结果时可打开“安全截图（严格重叠校验）”。
- 文字识别提供三个可独立装卸的模块。本地 OCR 使用 Windows 自带能力，对原图、高清放大、灰度增强和二值化候选结果择优，不携带模型；PP-OCR Tiny 使用较小的本机 ONNX 模型平衡速度与精度；PP-OCR Small 使用更大的本机 ONNX 模型优先处理小字、复杂背景和中英混排。三者都不上传图片，识别成功后会把文字填入选区旁的独立小窗，可继续编辑或一键复制；通常只需在“插件模块”页启用其中一个。
- 安装二维码扫描模块后，选区确认工具栏会增加“二维码”。完整框选二维码并点击即可在本机离线识别；成功后原始内容填入选区旁边的独立小窗，可编辑或复制。程序不会自动打开识别到的网址或执行二维码内容；模块可在“插件模块”页自由启用、禁用或永久删除。
- 录屏模块自带独立“录屏设置”分页，统一控制系统声音、麦克风、左键黄色半透明圆圈、30/60 FPS、2/4/8/12/20 Mbps 码率，以及录屏范围提示的实线、虚线或不显示；页面会实时估算 1 分钟、10 分钟和 1 小时的储存占用。安装模块后分页自动出现，卸载后自动消失；点击截图工具栏中的“录屏”后直接使用已保存参数。
- 可选录屏模块复用截图选区，录制 H.264 MP4、系统声音和麦克风；录制期间会在选区边缘显示默认虚线范围提示，该提示不会进入 MP4。左键黄色圆圈默认开启，录制现场和最终视频会同步显示，按住左键期间持续跟随鼠标移动，松开后短暂保留；设置关闭后两边都不显示。录屏工具栏在“矩形”左侧提供橙色专属“选择”按钮；开启后按钮切换为高对比绿色“✓ 选择中”并显示白色强调边框，左键可单选、`Ctrl` 多选、拖框选择、重叠轮选、移动和缩放编辑元素；拖框使用与截图模式一致的半透明蓝色填充和虚线边框，再次点击后按钮恢复橙色“选择”，鼠标恢复穿透并继续操作被录制程序。全屏录制中的元素移动、缩放、旋转、框选和画笔轨迹采用持久离屏画布与局部重绘，避免每次输入刷新整个透明全屏层，也避免拖动元素时透明层清空与重画之间闪烁。停止并保存后会直接结束本次截图会话，不再返回截图框；右下角会显示可点击的保存成功提示，点击即可在资源管理器中选中对应 MP4。取消录屏或保存失败时才会恢复截图框。录屏模块不再维护重复的形状或批注文档。默认下载完整包会预先安装录屏模块，仍可删除对应 DLL 或使用独立基础包以移除录屏功能。
- `Ctrl + C` 复制截图并结束，`Enter` 也可快速复制，`Ctrl + Z` 撤销。
- 单实例运行，避免重复注册热键。
- 支持从程序旁的 `Modules` 目录热加载、替换和拆卸扩展模块；正在进行的截图会话使用安全的模块快照。

## 构建和运行

源码构建要求 Windows 10/11 和 .NET 8 SDK。

```powershell
dotnet build .\ScreenshotTool.sln -c Release
dotnet run --project .\src\ScreenshotTool\ScreenshotTool.csproj
```

推荐发布为轻量 Windows x64 单文件。该版本体积最小，目标电脑需要安装 [.NET 8 Desktop Runtime](https://dotnet.microsoft.com/download/dotnet/8.0)：

```powershell
dotnet publish .\src\ScreenshotTool\ScreenshotTool.csproj -p:PublishProfile=LightweightWinX64 -o .\artifacts\lightweight-win-x64
```

如果目标电脑不能安装运行时，可发布压缩自包含版。它无需预装 .NET，但仍会包含完整 Windows Desktop Runtime，因此体积明显更大：

```powershell
dotnet publish .\src\ScreenshotTool\ScreenshotTool.csproj -p:PublishProfile=PortableCompressedWinX64 -o .\artifacts\portable-compressed-win-x64
```

WinForms 对程序集裁剪并不安全，本项目不使用 `PublishTrimmed` 冒险换取体积；轻量版与便携版执行完全相同的业务代码。

也可以运行下面的命令一次生成四种版本；`Bypass` 只对这一次 PowerShell 进程生效，不会修改系统执行策略：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\Publish-Release.ps1
```

脚本会检查轻量版不超过 5 MiB、便携压缩版不超过 90 MiB、依赖系统 .NET 的轻量完全版 ZIP 不超过 80 MiB，并把自带运行库的全插件完全版 ZIP 限制在 130 MiB 内，防止包体意外回退。`complete-lightweight-win-x64.zip` 和 `complete-portable-win-x64.zip` 都已把长截图、本地 OCR、二维码扫描、录屏 DLL、编码器和依赖安装到 `Modules`，但不会携带体积较大的 PP-OCR 模型；`complete-lightweight-full-win-x64.zip` 以轻量版为底包，额外预装 PP-OCR Tiny/Small 和全部插件；`complete-full-win-x64.zip` 提供相同插件集合并自带 .NET 运行库。脚本还会在压缩包旁自动生成 `LightShotCN-v<版本>-ready-to-run` 免解压运行目录，可直接启动其中的 `ScreenshotTool.exe`。脚本同时生成 `long-capture-addon-win-x64.zip`、`ocr-addon-win-x64.zip`、`paddle-ocr-tiny-addon-win-x64.zip`、`paddle-ocr-small-addon-win-x64.zip`、`qr-code-addon-win-x64.zip` 和 `screen-recording-addon-win-x64.zip`，便于用户从发布页按需组合或在永久删除后重新安装模块。本地 OCR 1.1.0、PP-OCR Tiny/Small 1.0.0 与二维码扫描模块 1.0.0 均需要轻截 1.11.0 或更高版本；录屏模块 1.7.0 需要轻截 1.10.0 或更高版本。

软件内更新依赖同一正式 Release 中的固定完整包文件名，以及 GitHub API 返回的 `sha256:` digest。发布新版本时必须同时上传轻量完整包和便携完整包；只改标签或发布说明而缺少完整包时，软件会明确停止安装。用户需要先运行一次包含“软件更新”页的版本，之后的正式版即可在软件内完成更新。

## 使用流程

1. 在左侧进入“截图设置”和“保存路径”，分别保存需要的配置；“截图设置”管理全局快捷键、开机自动启动、手动启动后最小化、截图前提示和主界面显示行为，“保存路径”中可选择日期时间、目录序号或图片内文字命名。安装录屏模块后会增加“录屏设置”，可选择声音、左键黄色圆圈、帧率、码率和范围提示；安装长截图模块后会增加“长截图”，可按需开启严格安全校验。
2. 按全局快捷键，或点击“立即截图”。
3. 移动鼠标自动高亮窗口，单击确认；也可按住左键拖动自由框选。需要整屏时把鼠标移到目标显示器并按一次 `Ctrl + A`；再按一次会扩展到全部显示器。
   安装录屏模块后，可在尚未添加截图批注时点击“录屏”；程序直接使用“录屏设置”页已保存的系统声音、麦克风、左键黄色圆圈、帧率和视频码率，不再弹出录屏选项窗。录制开始后仍使用截图时同一条编辑菜单栏；默认鼠标穿透到原程序，点击橙色“选择”可用左键编辑已有元素，点击矩形、画笔等工具可继续添加实时批注，再次点击当前按钮会回到鼠标穿透状态。
4. 从浮动工具栏选择标注工具；透明文字输入框会随内容自动扩展，当前“粗细”同时控制文字字号，输入期间调整会立即更新。编辑时按住 `Alt`，把鼠标移到虚线边框后使用左键拖动；使用 `Ctrl + Enter` 完成。完成后，单选文字并双击可再次编辑；文字、图片和各类绘制元素都统一使用 `Alt + 鼠标左键` 拖动；按住 `Alt` 在元素上滚动可旋转，按住 `Ctrl` 滚动可缩放鼠标下的元素。多选或 `Ctrl + A` 全选后，在组内任一元素上执行旋转或缩放会联动全部选中元素。重叠元素可在同一位置连续单击，从上到下轮换选中。松开 Alt 会自动恢复此前启用的绘图工具。单选可缩放元素后，可拖动四角或四边中点手柄；拖动期间按住 `Ctrl` 使用固定步长，普通拖动会按工具栏“吸附”开关对齐其他元素，快速双击 `Ctrl` 可切换吸附。按住 `Ctrl` 左键单击可逐个多选，也可直接按住鼠标左键拖出选择框批量选中相交元素；再用 `Alt + 鼠标左键` 拖动任一已选元素即可移动整组，`Delete` 删除整组。粗细范围、旋转步进、吸附距离和拖动步长由“图片修改”页设置。
5. 已添加标注后，空白处左键默认不会重新划定截图框；确实要放弃现有编辑并重划时，按 `Ctrl + W` 启动重新框选，再按住左键拖动选择新区域。
6. 按 `Esc` 会先退出当前编辑层；当前没有文字输入、元素选中、绘图工具或截图框调整状态时，再按 `Esc` 才关闭截图。
7. 按 `Ctrl + S` 保存到挂接文件夹并复制到剪贴板，或按 `Ctrl + C` 仅复制到剪贴板。截图或录屏保存成功后，可点击右下角提示直接打开目录并选中对应文件。
8. 以后发布新正式版后，可进入“软件更新”点击“检查更新”和“下载并安装”；校验通过后轻截会自动退出并重新打开。

## 工程结构

```text
src/ScreenshotTool/
├─ Abstractions/    服务接口，隔离 UI 与系统实现
├─ Application/     依赖组装入口
├─ Core/            设置、热键、桌面快照和几何模型
├─ Editing/         标注文档、工具类型和各类标注
├─ Infrastructure/  JSON 配置、全局热键、屏幕采集、PNG 保存
└─ Presentation/    组合式工作台、独立分页、主题、托盘和截图编辑覆盖层
src/ScreenshotTool.Contracts/
└─ ModuleContracts  外部模块唯一可依赖的稳定公共契约
src/ScreenshotTool.ScreenRecording/          可选录屏入口、透明批注层和录制控制
src/ScreenshotTool.ScreenRecording.Recorder/ 模块私有的 x64 音视频编码器进程
src/ScreenshotTool.Ocr/                      多路预处理的 Windows 本地 OCR 模块
src/ScreenshotTool.PaddleOcr/                PP-OCRv6 ONNX 共用识别引擎
src/ScreenshotTool.PaddleOcr.Tiny/           可独立装卸的 PP-OCR Tiny 入口模块
src/ScreenshotTool.PaddleOcr.Small/          可独立装卸的 PP-OCR Small 入口模块
src/ScreenshotTool.QrCode/                   可独立装卸的离线二维码扫描模块
modules/             运行时模块投放目录
```

UI 只依赖抽象接口；主工作台通过 `Shell + AppPage + 独立页面控件` 组合，不把分页逻辑堆进主窗体。录屏和长截图的设置控件位于各自模块程序集，通过通用设置页契约动态加入导航；模块不存在时没有对应入口或具体程序集依赖。系统 API 与文件 IO 集中在 `Infrastructure`；标注模型不依赖窗口控件。新增截图格式、配置存储方式或编辑工具时，可以在对应模块内扩展。

用户配置保存在 `%LocalAppData%\LightShotCN\Profiles\local.json`。JSON 包含配置版本、配置身份、快捷键、保存路径、图片命名规则、主界面截图可见性、贴纸移动模式、工具粗细范围和长截图安全开关；旧版 `settings.json` 会自动迁移且保留原文件。默认截图目录是“图片\轻截”，详细结构见 [用户偏好 JSON](docs/user-preferences.md)。

## 扩展模块

模块通过独立的 `ScreenshotTool.Contracts` 项目接入键盘、鼠标、预览、最终导出和可选设置页管线。每个模块独占程序旁 `Modules` 下的一个一级子文件夹；复制、替换或删除整个模块文件夹后无需重启主程序，设置入口会立即同步，新功能状态从下一次截图开始生效。设置工作台的“插件模块”页面会列出已安装模块，并允许启用、禁用或永久删除；禁用状态会跨重启保留，永久删除后需从发布页重新下载。托盘菜单仍提供“打开模块目录”和“重新加载模块”。

详细生命周期、示例和兼容性约束见 [模块化架构](docs/modular-architecture.md)。后续开发必须遵循根目录 [AGENTS.md](AGENTS.md) 中的组合与依赖边界。
