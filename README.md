# 轻截

一个使用 C# / .NET 8 / WinForms 编写的 Windows 桌面截图工具。程序可以常驻系统托盘，在后台监听全局快捷键；默认按 `Ctrl + Shift + X` 开始截图。

## 已实现功能

- 左侧分页、右侧内容的可视化工作台，集中提供信息粘贴、快捷键、复制、保存、编辑说明和截图浏览入口。
- 快捷键与保存路径使用独立设置页，可录入并持久化全局截图快捷键。
- 保存路径支持从资源管理器拖入单个文件夹，自动引用其完整路径；切换到新路径时会询问是否一并移动旧目录中的图片，同名图片不会被覆盖。
- “查看截图”页会读取保存目录中的最近截图，支持刷新、打开目录和双击查看原图。
- 可挂接截图保存文件夹，`Ctrl + S` 以时间命名保存 PNG，并自动复制最终图片到剪贴板。
- 保存后在桌面右下角显示约 3 秒成功通知，点击通知会打开目录并选中刚保存的图片。
- 关闭主窗口后继续在系统托盘后台工作，托盘菜单可截图、打开设置或退出。
- 支持多显示器虚拟桌面截图。
- 鼠标悬停时自动识别并高亮顶层窗口，左键单击即可确认该窗口。
- 按住左键并拖动会切换为自由框选；`Ctrl + A` 选取全部屏幕；`Esc` 取消。
- 选区确认后可拖动四条边和四个角微调范围，光标会显示对应方向的双向箭头。
- 选区确认后可在选区内部按住鼠标右键拖动整个截图框，按下时显示四向移动光标。
- “信息粘贴”设置可切换贴纸跟随模式：图片、粘贴文字和工具栏文字可随截图框一起移动，也可保持屏幕位置；固定模式中的越界内容只会暂时隐藏，不会被删除。
- 临时编辑工具：矩形、椭圆、箭头、画笔、透明背景文字、马赛克、颜色、线宽、撤销。
- 图片、文字、方框、椭圆、箭头、画笔线条和马赛克统一使用 `Alt + 鼠标左键` 拖动；Alt 是临时移动模式，松开后会自动回到此前启用的绘图工具。单选图片或绘制元素后，可用普通左键拖动四角手柄调整大小。
- 按住 `Ctrl` 并左键单击可添加或取消多个编辑元素的选中状态；使用 `Alt + 鼠标左键` 拖动任一已选元素会统一移动整组，按 `Delete` 可一次删除全部选中元素。
- 截图框内存在编辑元素且未启用绘图工具时，可按住鼠标左键拖出选择框，与选择框相交的编辑元素会被批量选中。
- `Ctrl + A` 会优先全选截图框中的全部编辑元素；元素已经全部选中后再次按下才扩展截图框到全屏。没有编辑元素时直接全屏。
- 截图框内已有编辑后，未选择工具时的左键不会误触重新框选；需要重划时按 `Ctrl + W` 启动重新框选，再按住左键拖动选择新区域。
- `Esc` 使用分层退出：先完成文字输入、取消元素选中、关闭绘图工具或结束截图框调整；仅在没有任何编辑状态时才关闭截图界面。
- 绘制工具按钮采用开关逻辑：单击启用，再次点击同一工具即可关闭。
- “图片修改”设置可配置编辑工具粗细上下限；截图时仅当鼠标悬停在“粗细”按钮上滚动，才会增减当前粗细。
- 选区确认后按 `Ctrl + V` 可粘贴剪贴板图片或文字；图片会作为贴纸自适应缩小，并支持拖动和四角等比缩放；粘贴文字与工具栏添加的文字也可选中拖动。
- 方框、椭圆、箭头、画笔线条和马赛克支持四角自由缩放，可分别调整宽度和高度；图片仍保持等比缩放。
- 选区确认后可点击“长截图”：程序保留原选区边框，并在旁侧打开始终位于滚动目标上方的独立实时预览窗；预览窗可拖动、调整大小和最大化，在预览画面上滚轮可缩放、左键拖动可平移查看。用户在蓝色选区内用滚轮向上或向下滚动时，程序会周期采集中间帧并双向拼接，回访已采集区域不会重复添加内容。点击“完成并编辑”后，长图直接回填到原截图选区并复用普通截图工具栏和编辑系统；滚轮以鼠标位置缩放，右键拖动选区内的长图画面，可继续使用矩形、椭圆、箭头、画笔、文字、马赛克、颜色、粗细、粘贴、撤销、多选、移动和缩放能力，再按原流程复制或保存。按 `Esc` 或点击“取消”会放弃本次长截图。“长截图”设置默认关闭安全校验，低置信度接缝或固定悬浮内容会按最可能位置继续拼接；需要严格结果时可打开“安全截图（严格重叠校验）”。
- `Ctrl + C` 复制截图并结束，`Enter` 也可快速复制，`Ctrl + Z` 撤销。
- 单实例运行，避免重复注册热键。
- 支持从程序旁的 `Modules` 目录热加载、替换和拆卸扩展模块；正在进行的截图会话使用安全的模块快照。

## 构建和运行

源码构建要求 Windows 10/11 和 .NET 8 SDK。

```powershell
dotnet build .\ScreenshotTool.sln -c Release
dotnet run --project .\src\ScreenshotTool\ScreenshotTool.csproj
```

推荐发布为轻量 Windows x64 单文件。该版本体积最小，目标电脑需要安装 [.NET 8 Desktop Runtime](https://dotnet.microsoft.com/download/dotnet/8.0)：

```powershell
dotnet publish .\src\ScreenshotTool\ScreenshotTool.csproj -p:PublishProfile=LightweightWinX64 -o .\artifacts\lightweight-win-x64
```

如果目标电脑不能安装运行时，可发布压缩自包含版。它无需预装 .NET，但仍会包含完整 Windows Desktop Runtime，因此体积明显更大：

```powershell
dotnet publish .\src\ScreenshotTool\ScreenshotTool.csproj -p:PublishProfile=PortableCompressedWinX64 -o .\artifacts\portable-compressed-win-x64
```

WinForms 对程序集裁剪并不安全，本项目不使用 `PublishTrimmed` 冒险换取体积；轻量版与便携版执行完全相同的业务代码。

也可以运行下面的命令一次生成两种版本；`Bypass` 只对这一次 PowerShell 进程生效，不会修改系统执行策略：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\Publish-Release.ps1
```

脚本会检查轻量版不超过 5 MiB、便携压缩版不超过 90 MiB，防止包体意外回退。

## 使用流程

1. 在左侧进入“快捷键设置”和“保存路径”，分别保存需要的配置；如需严格长截图，可在“长截图”页开启安全校验。
2. 按全局快捷键，或点击“立即截图”。
3. 移动鼠标自动高亮窗口，单击确认；也可按住左键拖动自由框选，需要整屏时按 `Ctrl + A`。
4. 从浮动工具栏选择标注工具；透明文字输入框会随内容自动扩展。编辑时按住 `Alt`，把鼠标移到虚线边框后使用左键拖动；使用 `Ctrl + Enter` 完成。完成后，文字、图片和各类绘制元素都统一使用 `Alt + 鼠标左键` 拖动；松开 Alt 会自动恢复此前的绘图工具。单选图片或绘制元素后，可用普通左键拖动四角手柄缩放。按住 `Ctrl` 左键单击可逐个多选，也可直接按住鼠标左键拖出选择框批量选中相交元素；再用 `Alt + 鼠标左键` 拖动任一已选元素即可移动整组，`Delete` 删除整组。需要微调工具粗细时，把鼠标停在“粗细”按钮上滚动；范围由“图片修改”页设置。
5. 已添加标注后，空白处左键默认不会重新划定截图框；确实要放弃现有编辑并重划时，按 `Ctrl + W` 启动重新框选，再按住左键拖动选择新区域。
6. 按 `Esc` 会先退出当前编辑层；当前没有文字输入、元素选中、绘图工具或截图框调整状态时，再按 `Esc` 才关闭截图。
7. 按 `Ctrl + S` 保存到挂接文件夹并复制到剪贴板，或按 `Ctrl + C` 仅复制到剪贴板。

## 工程结构

```text
src/ScreenshotTool/
├─ Abstractions/    服务接口，隔离 UI 与系统实现
├─ Application/     依赖组装入口
├─ Core/            设置、热键、桌面快照和几何模型
├─ Editing/         标注文档、工具类型和各类标注
├─ Infrastructure/  JSON 配置、全局热键、屏幕采集、PNG 保存
└─ Presentation/    组合式工作台、独立分页、主题、托盘和截图编辑覆盖层
src/ScreenshotTool.Contracts/
└─ ModuleContracts  外部模块唯一可依赖的稳定公共契约
modules/             运行时模块投放目录
```

UI 只依赖抽象接口；主工作台通过 `Shell + AppPage + 独立页面控件` 组合，不把分页逻辑堆进主窗体。系统 API 与文件 IO 集中在 `Infrastructure`；标注模型不依赖窗口控件。新增截图格式、配置存储方式或编辑工具时，可以在对应模块内扩展。

用户配置保存在 `%LocalAppData%\LightShotCN\Profiles\local.json`。JSON 包含配置版本、配置身份、快捷键、保存路径、贴纸移动模式、工具粗细范围和长截图安全开关；旧版 `settings.json` 会自动迁移且保留原文件。默认截图目录是“图片\轻截”，详细结构见 [用户偏好 JSON](docs/user-preferences.md)。

## 扩展模块

模块通过独立的 `ScreenshotTool.Contracts` 项目接入键盘、鼠标、预览和最终导出管线。将模块 DLL 放入程序旁的 `Modules` 目录即可自动加载；替换或删除 DLL 后无需重启主程序，新状态从下一次截图开始生效。托盘菜单提供“打开模块目录”和“重新加载模块”。

详细生命周期、示例和兼容性约束见 [模块化架构](docs/modular-architecture.md)。后续开发必须遵循根目录 [AGENTS.md](AGENTS.md) 中的组合与依赖边界。
